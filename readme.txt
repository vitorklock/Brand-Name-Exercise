Instruções gerais para criação, entrega e montagem do projeto:

O que se espera para o envio do projeto:
    - Ter uma semântica correta do html utilizado no projeto
    - Montar uma estrutura obedecendo as regras do desenvolvimento web
    - Utilizar os arquivos enviados dentro da pasta assets/images
    - Montar o projeto com a estrutura obedecendo as pastas já criadas, com o arquivo css customizado sendo colocado dentro da pasta assets/css, a nomenclatura do mesmo fica livre. Utilizando a página "index.html", já disponibilizada na raiz do projeto
    - O uso de um framework front-end, sendo o sugerido Bootstrap. Caso queira não utilizar nada, será aceito da mesma forma, porém dificultando o processo, já que toda a base da estrutura está montada no sistema de grids, tamanhos e classes do bootstrap.
    - Utilização correta dos headings e tags html para seus determinados funcionamentos
    - Visualização básica e funcional em dispositivos móveis (não necessitando criar outros elementos para o mobile)
    - Entrega do arquivo com as características citadas acima em um .zip

O que é legal de se ter "a mais" (porém totalmente opcional):
    - Configuração do favicon da página, com a imagem  disponibilizada
    - Configuração das meta tags de open graph para compartilhamento em redes sociais
    - Carregamento das imagens via lazy-load (loading="lazy")
    - Funcionamento do site em dispositivos móveis
    - Links da página inicial navegáveis para as sessões da página
    - Uso mínimo (ou não uso) de position: absolute na página

Para você produzir a página, deverá se basear no mockup disponível no seguinte link: https://www.figma.com/file/9cfTFTIqpYavsFIOntqz2N/Landing-Page-de-teste?node-id=2415%3A26437

Dentro da landing page você terá 3 bases simples para visualizar as informações necessárias: 
    - Template: contém a página como um todo, para você poder entender e visualizar o escopo total do projeto. Os tamanhos de container, colunas e base do grid é o padrão do bootstrap. Sendo o container utilizado pela classe container, as colunas baseadas no grid de 12 colunas, e o grid sendo responsivo (utilizando col-lg/col-md em locais com quebra de conteúdo em várias colunas)
    - Tipografia: dentro desse artboard você encontrará todas as informações de como utilizar as fontes do google, uma vídeo aula caso necessite, o link para a fonte no google fonts e uma separação com a base do código css para cada variação de fonte que você necessitar.
    - Cores: nessa artboard você terá todas as cores utilizadas no projeto, com seus nomes dentro da tela e as cores em hexadecimal para facilitar na criação
    - Para passar de uma artborad para outra, você pode utilizar os ícones de próximo e anterior na parte inferior da tela, ou navegar com o botão central do mouse. 
    - VOCÊ NÃO PRECISA ESTAR COM UMA CONTA LOGADA OU ATIVA PARA ABRIR O LINK E PROTÓTIPO

Informações sobre ícones e imagens:
    - Os ícones estão em formato SVG (scalable vector graphics) para não haver perda de qualidade e redimensionamento nos diversos browsers. A utilização é a mesma das imagens normais
    - As imagens devem ter o atributo alt setado (em tags img) para ter um score maior
    - Todas as imagens foram exportadas nos tamanhos e dimensões corretos para o uso em uma tela de no mínimo 350px, não se preocupe em redimensionar
